# deploy/ — Milestone 1: reproducible dev environment + stock CCH baseline

Runs the complete cross-chain stack locally and proves the **unmodified**
upstream CCH swap end to end. This is the regression baseline Bifrost is
measured against — no Bifrost features live here.

## Quick start

```bash
cp deploy/.env.example deploy/.env          # dev creds, never committed
docker compose -f deploy/docker-compose.dev.yml --env-file deploy/.env up -d --build
deploy/scripts/fund-regtest.sh              # regtest coins + LN channel hub->payee
deploy/scripts/smoke-cch.sh                 # stock CCH swap -> prints order at Success
```

## Topology (mirrors upstream fiber e2e CI, one compose service per process)

| Service | Role |
|---|---|
| `netbase` | anchor container owning the shared network namespace |
| `ckb` | CKB **dev chain** + contract deploy + wallet funding + `udt-init` (deploys wrapped-BTC xUDT, generates fnn configs) |
| `bitcoind` | regtest |
| `lnd-hub` / `lnd-payee` | LND pair ("ingrid"/"bob" upstream); hold-invoice support verified at smoke time |
| `fnn-client` | fiber node 1 — pays wBTC to the hub |
| `fnn-hub` | fiber node 3 — **cch service enabled** (config generated by udt-init) |
| `bifrostd` | placeholder (Milestone 2) |

Design notes:
- **Why a dev chain, not CKB testnet:** upstream's own e2e uses a local dev
  chain; public testnet cannot be self-funded (faucets, real wBTC), so
  `docker compose up` would not be reproducible. Decision logged 2026-07-14.
- **Why one shared network namespace:** the fiber test harness (compiled
  `udt-init`, every generated `config.yml`, the vendored lnd confs) assumes
  `127.0.0.1`. Sharing the namespace keeps upstream assets byte-identical =
  maximum regression fidelity. Services stay individually restartable.
- **No host-published ports.** All RPC stays inside the namespace
  (CLAUDE.md: never expose unauthenticated RPC beyond localhost). Scripts use
  `docker compose exec`.
- **Hex numeric params** (`0x5f5e100`): encode/decode lives in ONE module,
  `bifrostd/src/fnn/codec.ts` (unit-tested, bigint-only); shell scripts call
  it via `bifrostd/src/fnn/codec-cli.mjs`.
- `deploy/vendor/fiber-0.9.0-rc7/` is upstream source at tag v0.9.0-rc7 (tests/,
  config/, docker/ subset). The dev keys inside are upstream's throwaway test
  fixtures. Runtime state written there is gitignored.

## Reset

`deploy/vendor` is a BIND mount, not a named volume, and the containers write
chain/node state into it AS ROOT — so `docker compose down -v` alone doesn't
clean it, and a plain host-side `git clean -fdx deploy/vendor` fails with
"Permission denied" on the root-owned files (and, if you push through the
warnings, deletes `tests/nodes/deployer/dev.toml` and `.../deployer/config.yml`,
static upstream fixtures that nothing regenerates — recoverable only from the
pinned upstream tag). Use the shared helper instead, which cleans the bind
mount from inside a throwaway root container and leaves those fixtures alone:

```bash
source deploy/scripts/lib.sh && fresh_reprovision
```

`deploy/scripts/smoke-cch.sh --fresh` and `deploy/scripts/smoke-bifrost.sh --fresh`
call this for you.
